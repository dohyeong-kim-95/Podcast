# Podcast — PRD (Vercel Launch Edition)

## 1. 제품 정의

**Podcast**는 하루 동안 모아 둔 PDF와 이미지 소스를 바탕으로, 다음 날 아침에 개인 맞춤형 한국어 오디오 요약을 들을 수 있게 해주는 모바일 퍼스트 PWA다.

- 프로덕션 진입점: `https://podcast.bubblelab.dev`
- 핵심 가치: "어제 저장한 것을 오늘 아침에 듣는다"
- 출시 형태: 소규모 화이트리스트 사용자용 비공개 서비스
- 운영 원칙: 프론트엔드는 Vercel, 백엔드는 기존 FastAPI + Cloud Run을 유지한다

## 2. 이번 문서의 기준

이 PRD는 기존의 Firebase Hosting 중심 전제가 아니라, 다음 기준을 제품 요구사항으로 본다.

- 사용자는 `podcast.bubblelab.dev`에서 앱을 연다
- Google 로그인, PWA 설치, 푸시 알림, NotebookLM 재인증이 모두 해당 도메인에서 동작해야 한다
- 백엔드 재작성 없이 현재 FastAPI 서비스를 그대로 활용해 출시한다

## 3. 목표 사용자

- 본인 포함 최대 5명 내외의 화이트리스트 사용자
- 낮 동안 문서와 스크린샷을 많이 저장하지만, 다시 읽을 시간이 부족한 사용자
- 통근, 산책, 운동 시간에 전날 모은 정보를 오디오로 소비하고 싶은 사용자

## 4. 핵심 사용자 시나리오

### 4.1 일상 사용 플로우
1. 사용자는 낮 동안 `podcast.bubblelab.dev`에 접속해 PDF나 이미지 파일을 업로드한다.
2. 매일 06:40 KST 기준으로 전일 06:40부터 당일 06:40까지의 소스가 수집 윈도우로 묶인다.
3. 스케줄러가 백엔드를 호출해 사용자별 팟캐스트 생성을 시작한다.
4. 생성이 끝나면 푸시 알림이 도착한다.
5. 사용자는 앱에서 바로 재생하거나, 로컬에 다운로드해 듣는다.

### 4.2 재인증 플로우
1. NotebookLM 세션이 만료되었거나 만료 임박 상태가 되면 앱이 이를 보여준다.
2. 사용자는 설정 화면에서 새 탭 기반 재인증을 시작한다.
3. Browserless 원격 브라우저에서 Google 로그인을 완료한다.
4. 서버가 세션을 저장하고, 이후 생성 작업이 다시 가능해진다.

## 5. 출시 목표

### 5.1 제품 목표
- `podcast.bubblelab.dev`에서 앱 전체 흐름이 안정적으로 동작한다
- 업로드부터 재생까지 하루 1회 루프가 사용자 개입 없이 이어진다
- 세션 만료, 소스 없음, 생성 실패 같은 예외 상황이 사용자에게 명확히 보인다

### 5.2 운영 목표
- 프론트 배포는 Vercel만으로 관리한다
- API와 스케줄러는 Cloud Run + Cloud Scheduler로 유지한다
- Firebase Auth, Firestore, Storage, FCM 연동이 Vercel 커스텀 도메인 기준으로 정합성을 가진다

## 6. 출시 범위

### P0. 반드시 포함

#### 6.1 접근 제어와 로그인
- Google 로그인
- 이메일 화이트리스트 기반 접근 제어
- 로그인 직후 백엔드 검증

#### 6.2 소스 수집
- PDF 업로드
- PNG/JPEG/WEBP 업로드
- 이미지 업로드 시 서버에서 PDF 변환
- 당일 업로드 소스 목록 조회
- 개별 소스 삭제

#### 6.3 팟캐스트 생성
- 매일 06:40 KST 자동 생성
- 사용자의 메모리 설정을 생성 instructions에 반영
- 소스가 없으면 생성 스킵
- 실패 시 상태 기록과 수동 재생성 제공

#### 6.4 청취 경험
- 오늘의 팟캐스트 조회
- 인앱 재생
- 다운로드
- 청취 후 피드백 수집

#### 6.5 세션 관리
- NotebookLM 세션 상태 표시
- 새 탭 기반 재인증
- 만료/만료 임박 배너

#### 6.6 PWA 및 알림
- 홈 화면 설치 가능
- 서비스 워커 기반 기본 오프라인 진입
- 생성 완료 및 다운로드 리마인더 푸시

### P1. 출시 후 검토
- 과거 에피소드 히스토리
- URL 소스 직접 수집
- 트랜스크립트/텍스트 요약
- 운영 대시보드
- API 커스텀 도메인 분리

## 7. 비목표

이번 출시에서 다음은 하지 않는다.

- 백엔드를 Vercel Functions나 Next.js API Route로 이관
- 불특정 다수가 가입 가능한 공개 서비스로 확장
- 대규모 멀티테넌시, 팀 워크스페이스, 권한 레벨 설계
- 완전한 오프라인 재생 앱으로 확장

## 8. 성공 기준

- 화이트리스트 사용자 기준, 로그인 성공 후 메인 화면 진입 실패가 없어야 한다
- 소스가 있는 날의 자동 생성 성공률 90% 이상
- 생성된 팟캐스트의 대부분이 당일 재생 또는 다운로드된다
- NotebookLM 세션 만료 시 사용자가 스스로 재인증을 완료할 수 있다
- `podcast.bubblelab.dev` 기준으로 PWA 설치와 푸시 알림이 재현 가능하다

## 9. 제약과 리스크

- `notebooklm-py`는 비공식 라이브러리이므로 외부 변경에 취약하다
- NotebookLM 인증은 쿠키 기반이라 주기적 재인증이 필요하다
- Browserless와 FCM은 실제 배포 도메인/HTTPS 조건에서만 제대로 검증할 수 있다
- Vercel 배포에서는 Firebase Auth redirect helper 경로와 도메인 설정이 맞아야 한다
- 현재 백엔드는 Cloud Run 별도 도메인으로 동작하므로 CORS와 운영 환경변수 정합성이 중요하다
