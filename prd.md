# Podcast — PRD v1.0

## 1. 개요

**Podcast**는 하루 동안 수집한 PDF·스크린샷을 기반으로 매일 아침 7시에 10분짜리 한국어 AI 팟캐스트를 자동 생성해주는 모바일 퍼스트 웹앱이다.

- **핵심 가치**: "어제 내가 본 것들을, 오늘 아침 팟캐스트로 듣는다"
- **사용자**: 1~2명 (개인 프로젝트)
- **플랫폼**: 모바일 웹 PWA (Spotify 스타일 다크 UI)

## 2. 문제 정의

- 하루 동안 스크린샷·PDF로 캡처하는 정보가 많지만, 이를 종합적으로 소화할 시간이 부족함
- NotebookLM의 Audio Overview 기능은 강력하지만, 매일 수동으로 소스를 올리고 생성하기가 번거로움
- 통근·운동 시간에 "어제 내가 모은 정보"를 팟캐스트로 들으면 효율적

## 3. 사용자 시나리오

### 3.1 일상 플로우
1. **낮~밤**: 모바일에서 앱을 열어 PDF 업로드 또는 스크린샷 촬영/업로드
2. **06:00**: 소스 수집 컷오프 (전일 07:00 ~ 당일 06:00 업로드분)
3. **06:40**: Cloud Scheduler가 팟캐스트 생성 트리거
4. **~07:00**: 팟캐스트 생성 완료, PWA 푸시 알림
5. **아침**: 알림 탭 → 앱에서 팟캐스트 재생, 원하면 로컬 디바이스에 다운로드

### 3.2 재인증 플로우 (수 주에 1회)
1. 앱에서 "세션 만료" 배너 표시
2. "재인증" 버튼 탭
3. Browserless.io 원격 브라우저가 앱 내에서 열림 (iframe/팝업)
4. 사용자가 Google 로그인 수행
5. 쿠키 자동 추출 → 서버 암호화 저장 → 배너 사라짐

## 4. 기능 요구사항

### P0 (MVP)

#### 4.1 소스 업로드
- PDF 파일 업로드 (다중 선택)
- 스크린샷/이미지 업로드 → 서버에서 PDF 변환 후 NotebookLM 소스로 추가
- 업로드된 소스 목록 표시 (파일명, 업로드 시각, 썸네일)
- 소스 개별 삭제

#### 4.2 팟캐스트 생성
- 매일 06:40 KST 자동 트리거 (Cloud Scheduler → Cloud Run)
- 소스 윈도우: 전일 07:00 ~ 당일 06:00 업로드분
- notebooklm-py 플로우: 노트북 생성 → 윈도우 내 소스 전량 추가 → Audio Overview 생성 (한국어)
- 사용자 성향 메모리를 `generate_audio` instructions에 주입
- 자동 재시도 2회, 최종 실패 시 앱에서 수동 트리거 버튼 제공
- 소스가 0개인 날은 생성 스킵, 사용자에게 "오늘은 소스가 없어요" 알림

#### 4.3 팟캐스트 재생
- 인앱 오디오 플레이어 (재생/일시정지, 시크바, 배속 조절: 1x/1.5x/2x)
- 오늘의 팟캐스트만 서버에 보관 (다음 날 생성 시 이전 것 삭제)
- 로컬 디바이스 다운로드 버튼 (mp3)
- 백그라운드 재생 지원 (PWA)

#### 4.4 사용자 성향 메모리
- 사용자 프로필 설정 페이지에서 관심 분야, 선호 톤, 깊이 수준 등을 텍스트로 입력
- 팟캐스트 청취 후 간단 피드백 (좋았다/보통/별로) → 메모리에 누적
- 메모리 내용을 기반으로 `generate_audio` instructions 프롬프트 자동 구성
- Firestore에 사용자별 메모리 문서로 저장, 누적 업데이트

#### 4.5 NB 인증 관리
- Browserless.io 연동: 앱 내에서 원격 브라우저 세션 → Google 로그인 → 쿠키 자동 추출
- 쿠키 유효성 상태 표시 (유효 / 만료 임박 / 만료)
- 만료 시 앱 상단 배너로 재인증 유도
- 쿠키는 Firestore에 암호화 저장

#### 4.6 인증 & 접근 제어
- Firebase Auth (Google OAuth) 로그인
- 1~2명 화이트리스트 기반 접근 제어 (허용 이메일 목록을 환경변수로 관리)

### P1 (후순위)
- URL 소스 지원 (웹페이지 링크 직접 추가)
- 팟캐스트 히스토리 (과거 에피소드 다시 듣기)
- 소스별 요약 텍스트 표시
- 팟캐스트 트랜스크립트
- Firebase Storage 사용량 대시보드

## 5. 비기능 요구사항

| 항목 | 목표 |
|------|------|
| 팟캐스트 생성 시간 | 소스 업로드 완료 후 30분 이내 |
| 가용성 | 개인 프로젝트 수준 (SLA 불필요) |
| 비용 | Firebase Spark(무료) + Cloud Run 최소 사양 + Browserless.io 무료 티어 |
| 보안 | 화이트리스트 인증, NB 쿠키 암호화 저장 |

## 6. 제약사항

- **notebooklm-py 비공식 라이브러리**: Google 내부 RPC 역공학 기반. API 변경 시 서비스 중단 가능
- **쿠키 기반 인증**: Google 세션 쿠키 수 주~90일 주기 만료. 주기적 재인증 필수
- **Firebase Spark 한도**: Storage 5GB, Firestore 1GB. 소스 파일은 팟캐스트 생성 후 정리 필요
- **Audio Overview 생성 시간**: NotebookLM 서버 상태에 따라 5~15분 소요
- **Browserless.io 무료 티어**: 월 사용량 제한 있음 (재인증이 월 1-2회이므로 충분)
- **이미지→PDF 변환**: 서버사이드에서 Pillow/reportlab 등으로 변환. 텍스트 없는 이미지는 NotebookLM이 해석 가능한 수준에 의존

## 7. 성공 지표

- 하루 1회 이상 소스 업로드
- 팟캐스트 생성 성공률 90% 이상 (소스가 있는 날 기준)
- 생성된 팟캐스트의 80% 이상 당일 재생
